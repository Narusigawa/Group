### ТЕХНИЧЕСКОЕ ЗАДАНИЕ НА РАЗРАБОТКУ ПРИЛОЖЕНИЯ КОНДИТЕРСКОЙ "SWEET DELIGHTS"

### 1. Назначение приложения

  Разработать клиентское приложение для кондитерской "Sweet Delights", предназначенное для покупателей. Приложение должно предоставлять пользователям возможность просматривать ассортимент кондитерских изделий, получать подробную информацию о товарах, регистрироваться в системе, оформлять заказы и управлять личными данными. Интерфейс приложения выполняется в нежных, пастельных тонах, соответствующих тематике кондитерской.

### 2. Требования к интерфейсу
- Приложение должно иметь интуитивно понятный и приятный визуальный интерфейс в светлых тонах.
- Навигация по разделам осуществляется с помощью вкладок.
- Основные элементы интерфейса: карточки товаров, формы ввода данных, кнопки управления, информационные блоки.

### 3. Функциональные разделы приложения

3.1. Раздел "Каталог"**
- Является главным экраном приложения.
- Отображает список доступных товаров в виде карточек. Каждая карточка содержит изображение, название товара, его цену и кнопку для покупки.
- При клике на карточку товара открывается отдельный экран с подробным описанием товара, увеличенным изображением и кнопкой добавления в корзину.

3.2. Раздел "Личный кабинет"**
- Доступен только после авторизации пользователя.
- Отображает персональные данные пользователя: фамилию, имя, отчество, номер телефона, адрес электронной почты, адрес доставки.
- Предоставляет возможность редактирования всех перечисленных данных.
- Содержит раздел "История заказов", где отображается список предыдущих заказов с датами, суммами и статусами. При клике на заказ открывается подробная информация о его составе.
- Содержит кнопку для выхода из аккаунта.

3.3. Раздел "О нас"**
- Содержит общую информацию о кондитерской, её преимуществах и особенностях.

3.4. Раздел "Контакты"**
- Содержит контактную информацию: адрес, телефон, электронную почту.

### 4. Требования к системе авторизации и регистрации

- Доступ к личному кабинету и корзине возможен только после входа в систему.
- Для входа используются логин и пароль.
- Для новых пользователей предусмотрена форма регистрации, включающая следующие поля:
    - Фамилия
    - Имя
    - Отчество (необязательное поле)
    - Номер телефона
    - Адрес электронной почты
    - Адрес доставки
    - Логин
    - Пароль
- Система должна проверять уникальность логина и электронной почты при регистрации.

### 5. Требования к работе с корзиной и заказами

- Корзина доступна только авторизованным пользователям.
- Пользователь может добавлять товары в корзину из каталога или с детальной страницы товара.
- В корзине отображается список добавленных товаров с указанием названия, цены и выбранного количества.
- Пользователь может изменять количество товаров в корзине или удалять их.
- В корзине отображается итоговая сумма заказа.
- При оформлении заказа система создает запись о заказе с указанным адресом доставки и комментарием (при наличии). Корзина после оформления очищается.

### 6. Требования к данным и базе данных
- Вся информация о пользователях, товарах, категориях, корзине и заказах должна храниться в реляционной базе данных PostgreSQL.
- Таблицы базы данных должны быть связаны между собой для обеспечения целостности данных (например, заказы привязаны к пользователям, товары — к категориям).
- Информация о пользователях должна храниться и обрабатываться с возможностью последующего редактирования.
- Информация о товарах должна включать название, описание, цену, категорию, признак доступности и ссылку на изображение.
- Данные о заказах должны сохраняться с привязкой к пользователю и содержать перечень купленных товаров с их ценами на момент покупки (даже если товар будет удален из каталога, в истории заказов информация о нем должна сохраниться).
- При добавлении товара в корзину данные о нем должны временно храниться до момента оформления заказа или очистки корзины.

### 7. Технические требования к реализации
- Приложение должно быть разработано с использованием Windows Forms на платформе .NET.
- Для подключения и работы с базой данных PostgreSQL должна использоваться библиотека Npgsql.
- Приложение должно корректно обрабатывать ошибки подключения к базе данных, выводя понятные пользователю сообщения (например, "Не удалось подключиться к серверу, проверьте соединение").
- Интерфейс должен быть стабильным и не зависать при выполнении операций с данными.
- Все изменения данных (регистрация, редактирование профиля, оформление заказа) должны надежно сохраняться.
- Код приложения должен быть структурирован: отдельно работа с БД, отдельно модели данных, отдельно формы.
